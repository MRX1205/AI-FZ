# 高翠网宝塔快速部署

这个版本适合你现在的方式：

- 前端：宝塔 PHP/静态站点
- 后端：宝塔 Python 项目
- 数据库：Docker

这样最省事，因为数据库直接复用 `pgvector/pgvector:pg16`，不用在服务器里单独装向量扩展。

## 你现在日志里的问题

### 前端

前端访问日志本身没有报错，只说明：

- 宝塔已经把 `/` 返回出来了
- 浏览器能访问到 `merchant.lyhlz.cn`

### 后端

后端日志暴露了两个问题：

1. Python 虚拟环境没有安装依赖，所以报：

```text
ModuleNotFoundError: No module named 'fastapi'
```

2. 你把 Docker 环境变量也放进了后端 `.env`，之前后端会把这些当成错误字段。现在代码已经改成忽略额外变量了，可以继续共用一个 `.env`。

## 建议部署结构

服务器目录建议这样放：

```text
/www/wwwroot/merchant.lyhlz.cn/
├── index.html
├── assets/
├── favicon.svg
├── backend/
│   ├── app/
│   ├── alembic/
│   ├── alembic.ini
│   ├── requirements.txt
│   ├── .env
│   └── secrets/
└── deploy/
    ├── docker-compose.db.yml
    └── .env
```

## 第一步：上传文件

上传这几个内容到服务器：

- 前端静态文件：`FZ-front/dist/` 全部文件
- 后端包：`fz-server.tar.gz`
- 数据库 Docker 文件：`deploy/docker-compose.db.yml`
- 环境变量：把 `deploy/.env.production` 复制一份到服务器，分别放成：
  - `/www/wwwroot/merchant.lyhlz.cn/backend/.env`
  - `/www/wwwroot/merchant.lyhlz.cn/deploy/.env`
- 支付宝密钥文件：
  - `/www/wwwroot/merchant.lyhlz.cn/backend/secrets/alipay-app-private-key.pem`
  - `/www/wwwroot/merchant.lyhlz.cn/backend/secrets/alipay-public-key.pem`

后端解压：

```bash
cd /www/wwwroot/merchant.lyhlz.cn
tar xzf fz-server.tar.gz
mv fz-server backend
rm fz-server.tar.gz
mkdir -p backend/secrets deploy
```

## 第二步：启动数据库

```bash
cd /www/wwwroot/merchant.lyhlz.cn/deploy
docker compose --env-file .env -f docker-compose.db.yml up -d
docker compose --env-file .env -f docker-compose.db.yml ps
```

数据库启动后，后端会直接连：

- PostgreSQL：`127.0.0.1:55432`

这正好和你当前 `.env.production` 里的配置一致。

## 第三步：安装后端依赖

```bash
cd /www/wwwroot/merchant.lyhlz.cn/backend
python3.11 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
alembic upgrade head
```

## 第四步：宝塔 Python 项目启动命令

项目目录：

```text
/www/wwwroot/merchant.lyhlz.cn/backend
```

启动命令：

```bash
/www/wwwroot/merchant.lyhlz.cn/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 6141
```

如果宝塔里要填工作目录，就填：

```text
/www/wwwroot/merchant.lyhlz.cn/backend
```

## 第五步：宝塔站点 Nginx

站点根目录：

```text
/www/wwwroot/merchant.lyhlz.cn
```

Nginx 配置直接用项目里的：

```text
deploy/nginx.merchant.lyhlz.cn.conf
```

核心是三段：

- `/` 到前端静态文件
- `/api/` 反代到 `127.0.0.1:6141`
- `/uploads/` 反代到 `127.0.0.1:6141`

## 如果你想复用本地 Docker 数据

可以，但要注意：

- **Docker 镜像本身不带你的业务数据**
- 数据在 Docker **volume** 里

所以如果你想把本地已经有的商品、商家、向量数据一起带到服务器，迁移的不是镜像，而是数据库 volume 或数据库内容。

### 更稳的做法

继续复用服务器上的 `pgvector/pgvector:pg16` 镜像，只迁移你本地数据库内容。

### 最省事的做法

如果你本地也是 Docker PostgreSQL，可以把本地 PostgreSQL volume 打包后带到服务器，再挂到服务器容器上。

本地先找到 volume：

```bash
docker volume ls
```

停掉本地 PostgreSQL 容器后，把 volume 打包：

```bash
docker run --rm \
  -v 你的postgres卷名:/from \
  -v "$PWD":/to \
  alpine \
  sh -c "cd /from && tar czf /to/fz-postgres-data.tar.gz ."
```

服务器创建同名 volume 并恢复：

```bash
docker volume create fz_postgres_data
docker run --rm \
  -v fz_postgres_data:/to \
  -v "$PWD":/from \
  alpine \
  sh -c "cd /to && tar xzf /from/fz-postgres-data.tar.gz"
```

然后再启动：

```bash
cd /www/wwwroot/merchant.lyhlz.cn/deploy
docker compose --env-file .env -f docker-compose.db.yml up -d
```

## 上线检查顺序

先按这个顺序排：

1. `docker compose ps` 看数据库是否正常
2. `source venv/bin/activate && python -c "import app.main"` 看后端能否导入
3. `alembic upgrade head`
4. 启动宝塔 Python 项目
5. 访问：
   - `https://merchant.lyhlz.cn/api/health`
   - `https://merchant.lyhlz.cn/merchant/auth`

## 结论

对你现在这套环境，我建议这样定：

- 前端：继续宝塔静态站点
- 后端：继续宝塔 Python 项目
- 数据库：改用 Docker 跑 `pgvector`

这条路最少改动，也最接近你本地现状。
