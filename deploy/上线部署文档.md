# 高翠网宝塔部署文档

## 需要上传的文件

| 本地文件 | 上传到服务器 | 说明 |
|---|---|---|
| `FZ-front/dist/` 里所有文件 | `/www/wwwroot/merchant.lyhlz.cn/` | 前端静态文件 |
| `fz-server.tar.gz` | `/www/wwwroot/merchant.lyhlz.cn/` | 后端包 |

---

## 第 1 步：宝塔添加站点

1. 宝塔面板 → 网站 → 添加站点
2. 域名填 `merchant.lyhlz.cn`
3. PHP 版本选「纯静态」
4. 根目录 `/www/wwwroot/merchant.lyhlz.cn`
5. 提交

## 第 2 步：上传前端

宝塔 → 文件 → `/www/wwwroot/merchant.lyhlz.cn/`

1. 删除默认文件（index.html 等）
2. 上传 `FZ-front/dist/` 里的所有文件

## 第 3 步：上传后端

1. 上传 `fz-server.tar.gz` 到 `/www/wwwroot/merchant.lyhlz.cn/`
2. SSH 登录服务器解压：

```bash
cd /www/wwwroot/merchant.lyhlz.cn
tar xzf fz-server.tar.gz
mv fz-server backend
rm fz-server.tar.gz
```

3. 在 `/www/wwwroot/merchant.lyhlz.cn/backend/.env` 写入生产配置。
   可直接参考你本地的 `deploy/.env.production`，但要确认这些地址是线上值：
   - `SUPABASE_EMAIL_REDIRECT_TO=https://merchant.lyhlz.cn/merchant/auth`
   - `ALIPAY_NOTIFY_URL=https://merchant.lyhlz.cn/api/payments/alipay/notify`
   - `ALIPAY_RETURN_URL=https://merchant.lyhlz.cn/merchant/account/payment-result`
   - `FRONTEND_PUBLIC_BASE_URL=https://merchant.lyhlz.cn`
   - `BACKEND_PUBLIC_BASE_URL=https://merchant.lyhlz.cn`
   - `VITE_API_BASE_URL=https://merchant.lyhlz.cn`

解压后的目录结构：

```
/www/wwwroot/merchant.lyhlz.cn/
├── index.html              # 前端
├── assets/
├── favicon.svg
└── backend/                # 后端
    ├── app/                # Python 代码
    ├── alembic/            # 数据库迁移
    ├── alembic.ini
    ├── requirements.txt    # 依赖列表
    ├── .env                # 配置
    └── secrets/            # 支付宝密钥
```

## 第 4 步：服务器安装 Python 环境

```bash
apt update
apt install -y python3.12 python3.12-venv python3-pip
```

## 第 5 步：安装后端依赖

```bash
cd /www/wwwroot/merchant.lyhlz.cn/backend

# 创建虚拟环境
python3.12 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

## 第 6 步：配置数据库

注意：这个项目用了 `pgvector`，不能只装普通 PostgreSQL。

```bash
# Ubuntu 22.04 / Debian 系建议优先安装 PostgreSQL 16 + pgvector
apt update
apt install -y postgresql postgresql-contrib

# 如果你的系统仓库里有 pgvector，继续安装
# Ubuntu/Debian 常见包名可能是 postgresql-16-pgvector 或 postgresql-pgvector
# 先执行下面这条查看
apt-cache search pgvector

# 按搜索结果安装对应包，例如：
# apt install -y postgresql-16-pgvector
# 如果服务器没有这个包，建议直接使用 deploy/docker-compose.prod.yml 里的 pgvector/pgvector:pg16 数据库容器

# 启动
systemctl start postgresql
systemctl enable postgresql

# 创建数据库
sudo -u postgres psql << EOF
CREATE USER fz_user WITH PASSWORD 'fz_password';
CREATE DATABASE fz OWNER fz_user;
GRANT ALL PRIVILEGES ON DATABASE fz TO fz_user;
EOF

sudo -u postgres psql -d fz << EOF
CREATE EXTENSION IF NOT EXISTS vector;
EOF
```

## 第 7 步：初始化数据库表

```bash
cd /www/wwwroot/merchant.lyhlz.cn/backend
source venv/bin/activate
alembic upgrade head
```

## 第 8 步：测试启动

```bash
cd /www/wwwroot/merchant.lyhlz.cn/backend
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 6141
```

看到 `Uvicorn running on http://0.0.0.0:6141` 就成功了，Ctrl+C 停止。

## 第 9 步：后台运行

```bash
cat > /etc/systemd/system/fz-backend.service << 'EOF'
[Unit]
Description=FZ Backend API
After=network.target postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/www/wwwroot/merchant.lyhlz.cn/backend
Environment=PATH=/www/wwwroot/merchant.lyhlz.cn/backend/venv/bin:/usr/bin
ExecStart=/www/wwwroot/merchant.lyhlz.cn/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 6141
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable fz-backend
systemctl start fz-backend
```

## 第 10 步：宝塔 Nginx 配置

宝塔 → 网站 → `merchant.lyhlz.cn` → 配置文件，替换为：

```nginx
server {
    listen 80;
    server_name merchant.lyhlz.cn;
    root /www/wwwroot/merchant.lyhlz.cn;
    index index.html;

    client_max_body_size 50m;

    # 前端静态文件
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:6141;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        add_header Access-Control-Allow-Origin $http_origin always;
        add_header Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS, PATCH" always;
        add_header Access-Control-Allow-Headers "Authorization, Content-Type, Accept, Origin, X-Requested-With" always;
        add_header Access-Control-Allow-Credentials "true" always;

        if ($request_method = OPTIONS) {
            add_header Access-Control-Allow-Origin $http_origin always;
            add_header Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS, PATCH" always;
            add_header Access-Control-Allow-Headers "Authorization, Content-Type, Accept, Origin, X-Requested-With" always;
            add_header Access-Control-Allow-Credentials "true" always;
            add_header Access-Control-Max-Age 3600;
            add_header Content-Length 0;
            add_header Content-Type text/plain;
            return 204;
        }
    }

    # 文件上传
    location /uploads/ {
        proxy_pass http://127.0.0.1:6141;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

保存 → 重启 Nginx

如果你已经在宝塔里申请了 SSL 证书，记得打开强制 HTTPS。  
同时把后端 `.env` 里的这些地址保持为 `https://merchant.lyhlz.cn`：

- `SUPABASE_EMAIL_REDIRECT_TO`
- `ALIPAY_NOTIFY_URL`
- `ALIPAY_RETURN_URL`
- `FRONTEND_PUBLIC_BASE_URL`
- `BACKEND_PUBLIC_BASE_URL`
- `VITE_API_BASE_URL`
